## 🟡 주의 사항

### 🟡 Apache Tomcat: race condition in StandardContext.getServletContext() 패치 머지

- **CVE ID / 공지 번호**: 미배정 (CVE 없음, 내부 PR)
- **CVSS 점수**: 미배정
- **영향 범위**: main 브랜치 (미릴리즈, 최신 릴리즈 태그 11.0.21 이전)
- **자사 영향**: 확인 필요 (자사 버전 10.1.x — 10.1 브랜치 동일 패치 여부 미확인)
- **공식 링크**: https://github.com/apache/tomcat/pull/994
- **머지 일시**: 2026-04-23T13:37:05Z
- **변경 파일**:
  - `java/org/apache/catalina/core/StandardContext.java` (핵심 소스)
  - `test/org/apache/catalina/core/TestStandardContext.java`
  - `webapps/docs/changelog.xml`
- **취약점 내용**:  
  `context` 필드에 `volatile` 선언이 누락되고 `getServletContext()`에 동기화가 없어,  
  재로드(reload) 중 다수의 스레드가 경쟁적으로 `ApplicationContext` 인스턴스를 생성할 수 있음.  
  이로 인해 `TEMPDIR` 서블릿 컨텍스트 속성이 영구적으로 소실될 수 있음.
- **권장 조치**:
  - [ ] Tomcat 10.1 브랜치에도 동일 취약점 존재 여부 수동 확인
  - [ ] 자사 Tomcat 10.1 버전의 `StandardContext.getServletContext()` 코드 리뷰
  - [ ] Tomcat 10.1.x / 11.x 다음 패치 릴리즈 시 즉시 업그레이드 계획 수립
